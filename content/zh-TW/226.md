## 描述

HTTP `226 IM Used` 回應狀態碼在差異編碼的情境下，由伺服器設定，表示正在返回針對所收到 GET 請求的差異（delta）。IM 代表「實例操作」（Instance Manipulations），是生成差異的演算法術語。

在差異編碼中，伺服器以相對於給定基礎文件的差異（而非當前文件）回應 `GET` 請求。客戶端使用 `A-IM:` HTTP 標頭指定差異演算法，並使用 `If-None-Match:` 標頭提示伺服器上次取得的版本。伺服器生成差異後，以 `226` 狀態碼回應，並包含 `IM:` 和 `Delta-Base:` HTTP 標頭。

<aside class="info"><strong>注意：</strong>瀏覽器不支援 HTTP 差異編碼，此狀態碼由特定客戶端使用的自訂伺服器傳回。</aside>
