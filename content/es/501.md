## Descripción

El código de estado HTTP `501 Not Implemented` significa que el servidor no admite la funcionalidad necesaria para cumplir la solicitud.

Este estado también puede enviar una cabecera `Retry-After`, indicando al solicitante cuándo volver a comprobar si la funcionalidad está disponible.

`501` es la respuesta apropiada cuando el servidor no reconoce el método de solicitud y no puede admitirlo para ningún recurso. Los únicos métodos que los servidores deben admitir (y por tanto no deben devolver `501`) son `GET` y `HEAD`.

Si el servidor reconoce el método pero intencionalmente no lo admite, la respuesta apropiada es `405 Method Not Allowed`.
