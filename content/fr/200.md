## Description

Le code de statut HTTP `200 OK` indique que la requête a réussi. Une réponse 200 est mise en cache par défaut.

La signification d'un succès dépend de la méthode de requête HTTP utilisée :

- `GET` : la ressource a été récupérée et est transmise dans le corps du message.
- `HEAD` : les en-têtes de représentation sont inclus dans la réponse sans corps de message.
- `POST` : la ressource décrivant le résultat de l'action est transmise dans le corps du message.
- `TRACE` : le corps du message contient le message de requête tel que reçu par le serveur.

Le résultat d'une requête `PUT` ou `DELETE` réussie n'est souvent pas `200 OK` mais `204 No Content` (ou `201 Created` lorsque la ressource est téléchargée pour la première fois).
