## 描述

<aside class="warning"><strong>非官方：</strong>此 HTTP 状态码未由任何 RFC 指定。</aside>

HTTP `444 No Response` 是 nginx 专用的非官方状态码，表示 HTTP 连接已被关闭，不向客户端发送任何信息，包括此状态码本身。

当服务器生成 `444 No Response` 时，不会向客户端返回任何信息，直接关闭 HTTP 连接。此错误信息只会出现在 nginx 日志中，不会发送给客户端。它常用于处理恶意 HTTP 请求，例如包含非法 `Host` 头的请求。
