## Descripción

<aside class="warning"><strong>No oficial:</strong> Este código de estado HTTP no está especificado por ningún RFC.</aside>

HTTP `444 No Response` es un código de estado no oficial específico de nginx que indica que la conexión HTTP se ha cerrado. No se envía ninguna información al cliente, ni siquiera este código de estado.

Cuando se genera el código `444 No Response`, el servidor no devuelve ninguna información al cliente y cierra la conexión HTTP. Este mensaje de error solo aparece en los registros de nginx y no se envía al cliente. Es útil para tratar solicitudes HTTP maliciosas, como las que incluyen una cabecera `Host` ilegal.
