## Описание

<aside class="warning"><strong>Неофициально:</strong> Этот код состояния HTTP не определён ни одним RFC.</aside>

HTTP `444 No Response` — неофициальный код состояния, специфичный для nginx, указывающий на закрытие HTTP-соединения. Клиенту не отправляется никакая информация, включая сам этот код состояния.

При генерации кода `444 No Response` сервер не возвращает клиенту никакой информации и закрывает HTTP-соединение. Это сообщение об ошибке можно найти в журналах nginx, клиенту оно не отправляется. Оно полезно для обработки вредоносных HTTP-запросов, например содержащих недопустимый заголовок `Host`.
