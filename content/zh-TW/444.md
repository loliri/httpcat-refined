## 描述

<aside class="warning"><strong>非官方：</strong>此 HTTP 狀態碼未由任何 RFC 指定。</aside>

HTTP `444 No Response` 是 nginx 專用的非官方狀態碼，表示 HTTP 連線已被關閉，不向客戶端傳送任何資訊，包括此狀態碼本身。

當伺服器生成 `444 No Response` 時，不會向客戶端返回任何資訊，直接關閉 HTTP 連線。此錯誤資訊只會出現在 nginx 日誌中，不會傳送給客戶端。它常用於處理惡意 HTTP 請求，例如包含非法 `Host` 標頭的請求。
